Transparência

Por que self-hosted?

Seus dados financeiros são os mais sensíveis que existem. Eles não pertencem a nenhum servidor além do seu.

Segurança Física

Seus dados ficam em um arquivo SQLite no seu próprio disco. Nenhuma credencial, extrato ou posição trafega pela internet.

  • Sem autenticação em nuvem
  • Sem sync automático externo
  • Backup sob seu controle

Performance Local

Sem latência de servidor remoto. Toda a lógica roda localmente em milissegundos. Cotações e dados externos são opcionais e cacheados.

  • Funciona 100% offline
  • Cache local de cotações
  • Sem dependência de API paga

Open Source

Código auditável linha a linha. Sem telemetria, sem analytics, sem pixel de rastreamento. Você vê exatamente o que roda na sua máquina.

  • Zero telemetria
  • Sem dependências proprietárias
  • Modificável e extensível
FAQ

Perguntas frequentes

Dúvidas comuns sobre funcionamento, privacidade e instalação.

Todos os dados ficam em arquivos SQLite dentro do diretório configurado em FIN_DATA_DIR (padrão: _local_data/). Basta copiar essa pasta para outro local ou nuvem de sua preferência. Não existe lock-in — o formato SQLite é aberto e pode ser lido por qualquer ferramenta compatível.
Não há conexão com bancos ou corretoras. As únicas chamadas externas são opcionais: cotações de ações via yfinance e brapi.dev, e taxa CDI via API do Banco Central. Todas são cacheadas localmente. O app funciona normalmente offline — apenas cotações em tempo real ficam indisponíveis.
Sim. Ao abrir o módulo de Portfólio, as cotações são buscadas automaticamente e armazenadas em cache por 5 minutos. Dados fundamentalistas e CDI têm cache de 1 hora. Nenhuma chave de API é necessária para uso básico — o token da brapi é opcional e melhora os limites de requisição.
Nunca. O app não tem telemetria, não registra eventos de uso e não transmite nenhuma informação sobre sua carteira, extratos ou metas para servidores externos. As únicas requisições de rede são as consultas de cotações públicas descritas acima — e mesmo essas são opcionais.
A forma mais simples é via Docker Compose: clone o repositório, rode docker compose up e acesse /app/. Alternativamente, instale as dependências com pip install -r requirements.txt e execute streamlit run app.py. O banco de dados é criado automaticamente na primeira execução.
